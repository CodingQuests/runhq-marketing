## X / Twitter (short, punchy, thread-friendly)

**Post 1 — hook**
> The bottleneck in shipping software isn't writing code anymore.
>
> Your agent writes a feature in 4 minutes. Then it sits for 2 days waiting for review, a preview env, and someone to ship it safely.

**Post 2 — thread (4 tweets)**
1. We use AI coding agents every day. Here's what nobody tells you: the agent finishing the code is the *easy* part. 🧵
2. The hard part is everything after: turning feedback into a clear task, reviewing what the agent wrote, getting a preview up, shipping without breaking prod.
3. Most teams duct-tape Linear + GitHub + Vercel + a spreadsheet around their agents. It doesn't scale.
4. So we built RunHQ — an operations layer for coding agents. Feedback in → reviewed PR → preview → shipped. Early access open, reply if you want in.

**Post 3 — contrarian**
> Hot take: "AI will write all our code" misses the point. Writing was never the bottleneck. Reviewing, testing, and shipping safely is. That's the part that needs the most help right now.

## Posting rules of thumb
- **Disclose** when you're the maker (especially on Reddit). It builds trust and avoids bans.
- **One idea per post.** Reuse the messaging angles A–F so you learn which resonates organically too.
- **Reply fast** to every comment/DM in the first hour — that's where demos get booked.
- **Repurpose:** a strong X thread → LinkedIn post → Reddit discussion, adapted to each culture.
